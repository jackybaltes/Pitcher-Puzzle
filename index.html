Typical Pitcher Puzzle

    You have one 16 liter pitcher full of water, one empty nine liter pitcher, and one empty 7 liter pitcher. Using nothing but these three pitchers, can you divide the water evenly so that the 16 liter and nine liter pitchers are each holding exactly 8 liter of water?
  • We can pour water from jug J1 to jug J2 until
    • Jug J1 is empty
    • or
    • Jug J2 is full
  • We can empty a pitcher by throwing away the water
    • This action is not possible for all pitcher puzzles
  • We can fill a pitcher from a tap
    • This action is not possible for all pitcher puzzles
  • state representation
  • successor function
  • is_goal function
def heuristic_search( initial, successor_func, is_goal_func, heur_func, node_limit = 1000, depth_limit = None, ax = None ):
    queue = [ ( initial, heur_func(initial), 0, [initial] ) ]
    num_nodes = 1
    max_queue = 0
    sol = None
    visited = [ ]
    frames = []

    #print("Search started", queue)
    while( len( queue ) > 0 ):
        # queue = queue[0:-1]
        #print("Q", queue)
        if ( len(queue) > max_queue ):
              max_queue = len(queue)
        current, current_h, current_g, chist = queue.pop()
        visited.append( current )
        if ax is not None:
          #print('visited', visited )
          frame = drawEnvironment( ax, env, visited )
          frames.append( frame )

        #print('   ' * len(chist), current, '[', chist, ']' )
        #print("Num Nodes", numNodes)
        #print( len(chist), depthLimit, printState( current ) )
        if is_goal_func( current ):
            sol = chist
            break
        if ( depth_limit is None ) or ( len(chist) <= depth_limit ):
            #print("current state", current, chist )
            #print("queue", queue)
            children = successor_func( current )
            #print("children", children)
            for c in children:
                if c not in chist:
                    #print("Add child", printState(c))
                    h = heur_func( c )
                    pos = 0
                    for i in range(len(queue)-1, -1, -1 ):
                      if ( queue[i][1] >= h ):
                        pos = i + 1
                        break
                    queue.insert(pos, (c, h, 0, chist + [c] ))
                    num_nodes = num_nodes + 1
                    if ( num_nodes > node_limit ):
                        print("search terminated with num_nodes", num_nodes )
                        return ( num_nodes, max_queue, None )
        # else:
        #   print("Depth cutoff", len(chist), ">", depth_limit )
    return ( num_nodes, max_queue, sol, frames )

Heuristic Function for the Pitcher Puzzle

  • Difference between the current state and the goal state
  • Sum of squared differences
  • Sum of absolute differences

Bad Heuristic Functions

  • Finding a good heuristic function is extremely important in AI problems
  • Puzzles are particularly challenging for heuristic function, because puzzles are created in such a way that human intuition is mislead
  • In the previous trace the heuristic function does not provide much guidance since you can see it change dramatically during the solution
  • A good heuristic that measures error or distance to the goal should decrease monotonically while approaching the goal

  • Investigate if adding empty and fill commands will make the puzzle solution shorter

def successors2_func( state ):
  j1,j2,j3 = state
  children = []
  for frm in range( len(state) ):
    for to in range( len(state) ):
      
      if frm != to:
          c_frm = state[frm][CONTENT]
          c_to = state[to][CONTENT]

          if c_to + c_frm < state[to][CAPACITY]:
            pour = c_frm 
          else:
            pour = ( state[to][CAPACITY] - state[to][CONTENT] )

          if pour > 0:
            c = [ [ j[0], j[1] ] for j in state ]
            c[frm][CONTENT] = c[frm][CONTENT] - pour
            c[to][CONTENT] = c[to][CONTENT] + pour
            children.append(c)
    # Empty action
    if state[frm][CONTENT] > 0:
      c1 = [ [ j[0], j[1] ] for j in state ]
      c1[frm][CONTENT] = 0
      children.append(c1)

    # Fill action
    if state[frm][CONTENT] < state[frm][CAPACITY]:
      c2 = [ [ j[0], j[1] ] for j in state ]
      c2[frm][CONTENT] = c2[frm][CAPACITY]
      children.append(c2)

  return children

Impact of Branching Factor

  • Increasing the branching factor can greatly increase the size of the search space
  • Breadth First Search expanded 330 nodes for this puzzle and found the optimal solution with 15 moves
  • The max. branching factor was 6 in the puzzle without emptying and filling actions
  • Worst case estimate of 470184984576 for Breadth First search
  • Average branching factor: 1.47
  • The max. branching factor was 6+6 in the puzzle without emptying and filling actions
  • Worst case estimate of 15407021574586368 for Breadth First search. 32768.0 times larger

Problems of Adding Empty and Fill Actions

  • Impact of Empty Action: We start with 16 liters and we need 8+8 liters at the end. So if we ever empty a jug with water, then from then onwards it will be hard to solve the puzzle
  • Impact of Fill Action: We now have too much water
  • Adding those actions, the system has to figure out that the only way to a solution is to have matching empty and fill actions

Problems of Adding Empty and Fill Actions

  • In modern AI, avoiding is the crux of AI
  • Heuristic functions are powerful and versatile tools to reduce the size of the search space
  • Heuristic functions can be based on many difference features: distance to the goal, shape of the solution, Go the shape of a group was considered an important feature for the search, ...
  • Heuristic functions are a powerful way of including all kinds of domain knowledge
